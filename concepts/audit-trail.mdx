Every call to `guard()` or `check()` produces an **audit event**. The SDK calls `/sdk/agent_check`, the backend decides, the decision is logged, and the call returns. If the action runs, `guard()` follows up with `/sdk/agent_report` to attach the outcome and the on-chain signature.

## What gets written

| Field | Meaning |
| --- | --- |
| `agent_pda` | Which agent acted. |
| `action` | The `ActionDescriptor` (type, asset, amount, target). |
| `allowed` | `true` if policy allowed, `false` if denied. |
| `reason_code` | Machine-readable: `over_per_tx_cap`, `program_not_in_scope`, `agent_paused`, etc. |
| `tx_signature` | The Solana signature if the action landed. Attached by `report()`. |
| `created_at` | UTC timestamp. |
| `merkle_root_committed` | Set after the hourly anchor lands. Lets you verify on-chain. |

## On-chain anchoring

Every hour, the backend takes all audit rows that haven't been anchored yet, computes their Merkle root in a canonical order, and calls `commit_audit_root` on the operator account. The root is readable from the operator's `lastAuditRoot` field on Solscan.

This means:

- An auditor can fetch the same events from Altheia, hash them in the same canonical order, and compare the root to what's on-chain. If they don't match, something was rewritten.
- Altheia cannot rewrite history without breaking the chain of roots. Past roots reference past roots.

The hourly cadence is a current implementation choice, not a contract. Future versions may anchor more frequently.

## Denials are loud

Every denial writes a row in the chronicle and a Solana memo on the agent's PDA at the same time. So a denied action is visible in three places:

- The dashboard chronicle.
- A Slack webhook, if you wired one up.
- Solscan, under the agent's account.

## How to read the trail

| Surface | What you see | Auth |
| --- | --- | --- |
| **Dashboard** | The agent's chronicle, paginated and filterable. | Operator session (Phantom). |
| **Solscan** | Memo events for denials, plus `commit_audit_root` events on the operator account. | None. Public chain data. |

A first-class HTTP API for events is on the roadmap. For now, the dashboard is the canonical reader.
