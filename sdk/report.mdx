---
title: "report()"
description: "Attach an outcome to an audit row written by check()."
---

```ts
report(event: {
  action: ActionDescriptor;
  outcome: "success" | "failure";
  audit_event_id?: string;
  detail?: string;
  tx_signature?: string;
}): Promise<void>
```

`guard()` calls `report()` for you. You only call it directly when you split the flow: do the policy decision yourself with [`check()`](/sdk/check), then execute and report separately.

## When you'd use it directly

```ts
const decision = await altheia.check(action);
if (!decision.allowed) return;

try {
  const sig = await runIt();
  await altheia.report({
    action,
    outcome: "success",
    audit_event_id: decision.audit_event_id,
    tx_signature: sig,
  });
} catch (err) {
  await altheia.report({
    action,
    outcome: "failure",
    audit_event_id: decision.audit_event_id,
    detail: err instanceof Error ? err.message : String(err),
  });
  throw err;
}
```

Pass `audit_event_id` from the `check()` decision. That way the outcome attaches to the same row instead of creating a duplicate.

## Parameters

| Field | Type | Notes |
| --- | --- | --- |
| `action` | `ActionDescriptor` | Same shape you used in `check()`. |
| `outcome` | `"success" \| "failure"` | Whether the action ran. |
| `audit_event_id` | `string` | UUID from the `check()` decision. Optional. |
| `detail` | `string` | Free-form notes. Surfaced in the chronicle. |
| `tx_signature` | `string` | Solana signature. Becomes a Solscan link in the chronicle. |

## Best-effort

`report()` never throws. If the network is unreachable, it logs a warning and resolves. Audit completeness is backstopped by the backend's reconciliation cron: it watches the agent's session key for transactions and fills in missing reports from chain data.

## Don't double-report

`guard()` already reports. Calling `report()` after `guard()` for the same action creates a duplicate row.
