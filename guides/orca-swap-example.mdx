A runnable agent that swaps on Orca Whirlpools, wrapped by `altheia.guard()`. Two real on-chain swaps. Four denials. One pause, one unpause, one revoke. End-to-end proof of the operator-control loop.

## What this proves

| Claim | Where it's verified |
| --- | --- |
| Cap denial fires before signing | Scenario 1. Wallet never signs. |
| Program filter denies wrong DEX | Scenario 2. Jupiter tx never built. |
| Real on-chain swap when policy allows | Scenario 3. Solscan devnet tx link. |
| Pause stops the agent mid-loop | Scenario 4. Same action that worked is now denied. |
| Unpause brings the agent back | Scenario 5. Second real swap. |
| Revoke is permanent | Scenario 6. Same action denied, no recovery. |
| Audit captures every path | Chronicle: 2 allowed + 4 denied + 3 lifecycle rows. |

## The six scenarios

| # | Setup | Expected |
| --- | --- | --- |
| 1 | Swap 0.5 SOL on Orca (over per-tx cap) | DENIED `over_per_tx_cap` |
| 2 | Swap 0.1 SOL on Jupiter v6 (not allowlisted) | DENIED `program_not_in_scope` |
| 3 | Swap 0.05 SOL on Orca (within cap, allowed program) | ALLOWED, real tx on devnet |
| 4 | Operator pauses → retry scenario 3 | DENIED `agent_paused` |
| 5 | Operator unpauses → retry scenario 3 | ALLOWED, second real tx |
| 6 | Operator revokes → retry scenario 3 | DENIED `agent_revoked` (permanent) |

Scenarios 4, 5, 6 pause for an Enter keystroke. Switch to the dashboard, click the lifecycle action, wait for Phantom to confirm, hit Enter, and the script retries the same action.

## Setup

### 1. Register the agent with this policy

In the dashboard, register an agent (see [Register an agent](/guides/register-agent) for the full walk-through). Use this policy:

| Field | Value |
| --- | --- |
| `asset_caps.SOL` | `{ max_per_tx: 0.2, max_per_day: 1.0 }` |
| `asset_caps.USDC` | `{ max_per_tx: 100, max_per_day: 500 }` |
| `allowed_programs` | `["whirLbMiicVdio4qvUfM5KAg6Ct8VwpYzGff3uctyCc", "TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA"]` |
| `blocked_destinations` | `[]` |

Save all three secrets from the reveal modal: `agentPda`, `sessionKeyBase58`, `apiKey`. Plus the session key pubkey.

### 2. Airdrop SOL to the session key

```bash
solana airdrop 1 <sessionKeyPubkey> --url devnet
```

You need ~0.2 SOL for the two real swaps plus gas. If `solana airdrop` rate-limits, use [faucet.solana.com](https://faucet.solana.com/).

### 3. Clone, configure, run

```bash
git clone https://github.com/altheia-xyz/sdk.git
cd sdk/examples/orca-swap-agent
pnpm install
cp .env.example .env
# Edit .env with agentPda, sessionKeyBase58, apiKey from the reveal modal
pnpm demo
```

`.env` needs:

```bash
ALTHEIA_AGENT_PDA=
ALTHEIA_API_KEY=
ALTHEIA_SESSION_KEY=
ALTHEIA_BACKEND=https://api.altheia.xyz
SOLANA_RPC_URL=https://api.devnet.solana.com
WHIRLPOOL_ADDRESS=3KBZiL2g8C7tiJ32hTv5v3KM7aK9htpqTw4cTXz1HvPt
```

## The guard call

```ts
const sessionKey = Keypair.fromSecretKey(bs58.decode(process.env.ALTHEIA_SESSION_KEY!));

const altheia = new Altheia({
  agentPda: AGENT_PDA,
  apiKey: API_KEY,
  endpoint: BACKEND,
  failureMode: "closed", // backend unreachable → halt, no fail-open
  timeoutMs: 5000,
});

const sig = await altheia.guard(
  { type: "swap", asset: "SOL", amount: amountSol, target: ORCA_WHIRLPOOL_PROGRAM },
  async () => {
    const tx = await buildOrcaSwap(amountSol, SOL_MINT, DEV_USDC_MINT);
    return await tx.buildAndExecute(); // session key signs
  },
);
```

`failureMode: "closed"` is the right call for agents that move money. If the Altheia backend goes down, denials still halt instead of letting the agent swing free.

## What the run looks like

Scenarios 1, 2 deny before any signature. The session key never touches them.

Scenario 3 lands a real tx on devnet. Solscan link appears in the output.

Scenarios 4-6 wait for you to click in the dashboard (Pause, Unpause, Revoke), then retry the same swap to show the policy state changed.

The script ends with an invariant check: the underlying `orcaSwap()` function should have been called exactly twice (scenarios 3 and 5). If it ran any other count, the kill switch failed and the script prints a loud failure banner.

## After the run

Open `altheia.xyz/agents/<agentPda>`. The chronicle has:

- 2 `action_allowed` rows with Solscan signatures
- 4 `action_denied` rows with `reasonCode` values matching the table above
- 3 lifecycle rows: `agent_paused`, `agent_unpaused`, `agent_revoked`

That's the operator-control loop, end to end.
