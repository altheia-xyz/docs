---
title: "Quickstart"
description: "Zero to a guarded action on devnet in ten minutes."
---

You'll register an agent in the dashboard, fund its session key from a devnet faucet, then run a Node script that calls `altheia.guard()` against the real backend. By the end you'll see one allow and one deny in the chronicle.

## Prerequisites

<Steps>
  <Step title="Phantom wallet, switched to devnet">
    Install [Phantom](https://phantom.app/). Open Settings, **Developer Settings**, set network to **Devnet**.
  </Step>
  <Step title="Devnet SOL in your operator wallet">
    [faucet.solana.com](https://faucet.solana.com/) → paste your Phantom address → airdrop 1 SOL. You'll spend ~0.01 SOL on registration + lifecycle signatures.
  </Step>
  <Step title="Node 20+ and pnpm">
    `node -v` should say 20 or higher. `npm i -g pnpm` if you don't have it.
  </Step>
</Steps>

## 1. Sign in

Open [altheia.xyz/dashboard](https://altheia.xyz/dashboard). Click **Connect wallet**, sign the challenge in Phantom. Your wallet becomes the **operator**. On first sign-in an `OperatorAccount` PDA is created on Solana.

## 2. Register your first agent

Click **+ Register agent**.

<Frame>
  <img src="/images/policy.png" alt="Policy section in the dashboard, same fields you'll fill on the register form" />
</Frame>

Fill the form:

- **Name.** Anything memorable. Shows up in audit rows.
- **Caps.** Start with SOL `max_per_tx 0.2`, `max_per_day 1`. You can edit later.
- **Allowed programs.** Pick **Orca Whirlpools** from the catalog. Add **Token Program** too if you'll need it.

Submit. Phantom pops up once. Sign.

<Warning>
  After the tx lands, the dashboard shows a **reveal modal with three values shown once**:

  1. `agentPda`. The agent's on-chain identity. Public, also visible on Solscan.
  2. `sessionKeyBase58`. The agent's private keypair, generated in your browser. **Backend never had it. Lose it and you must revoke + re-register.**
  3. `apiKey`. The SDK's bearer token. Rotatable from the agent detail page if you lose it.

  Copy all three into a password manager **before you close the modal**. The session keypair is the one you cannot recover; the other two are convenience.
</Warning>

## 3. Fund the session key

The session key is a fresh Solana keypair with zero balance. It pays its own gas and provides the SOL the agent will spend. The reveal modal shows its **public** address.

```bash
solana airdrop 0.5 <sessionKeyPubkey> --url devnet
```

Or paste the pubkey into [faucet.solana.com](https://faucet.solana.com/) if the CLI rate-limits.

## 4. Install the SDK

```bash
mkdir altheia-hello && cd altheia-hello
pnpm init
pnpm add @altheia-xyz/sdk @solana/web3.js bs58
pnpm add -D tsx
```

Create `.env`:

```bash
ALTHEIA_AGENT_PDA=<paste from reveal modal>
ALTHEIA_API_KEY=<paste from reveal modal>
ALTHEIA_SESSION_KEY=<paste sessionKeyBase58 from reveal modal>
```

## 5. Wire `altheia.guard()`

Create `hello.ts`:

```ts hello.ts
import { Altheia, PolicyDeniedError } from "@altheia-xyz/sdk";
import { Connection, Keypair, SystemProgram, Transaction, LAMPORTS_PER_SOL } from "@solana/web3.js";
import bs58 from "bs58";

// 1. Load the session key. Backend never had this secret; it lives in your env.
const sessionKey = Keypair.fromSecretKey(bs58.decode(process.env.ALTHEIA_SESSION_KEY!));

// 2. Construct the Altheia client. agentPda + apiKey is all it needs.
const altheia = new Altheia({
  agentPda: process.env.ALTHEIA_AGENT_PDA!,
  apiKey: process.env.ALTHEIA_API_KEY!,
  failureMode: "closed",
  timeoutMs: 5000,
});

// 3. Wrap a real on-chain action. The session key signs the underlying tx.
const conn = new Connection("https://api.devnet.solana.com", "confirmed");
const recipient = sessionKey.publicKey; // send to self for the demo

async function sendSol(lamports: number) {
  return altheia.guard(
    { type: "transfer", asset: "SOL", amount: lamports / LAMPORTS_PER_SOL, target: recipient.toBase58() },
    async () => {
      const tx = new Transaction().add(
        SystemProgram.transfer({ fromPubkey: sessionKey.publicKey, toPubkey: recipient, lamports }),
      );
      const sig = await conn.sendTransaction(tx, [sessionKey]);
      await conn.confirmTransaction(sig, "confirmed");
      return sig;
    },
  );
}

// 4. One allowed, one denied.
try {
  const sigOk = await sendSol(0.01 * LAMPORTS_PER_SOL);
  console.log("allowed, signature:", sigOk);
} catch (err) {
  if (err instanceof PolicyDeniedError) console.log("unexpected deny:", err.reasonCode, err.reason);
}

try {
  await sendSol(5 * LAMPORTS_PER_SOL); // over the 0.2 SOL cap
} catch (err) {
  if (err instanceof PolicyDeniedError) console.log("denied as expected:", err.reasonCode, err.reason);
}
```

Run it:

```bash
pnpm tsx --env-file=.env hello.ts
```

You should see one `allowed` line with a real signature, then one `denied as expected: over_per_tx_cap`.

## 6. Verify in the dashboard

Open `altheia.xyz/agents/<agentPda>`. The chronicle has two rows: an `action_allowed` with a Solscan link, and an `action_denied` with reason `over_per_tx_cap`.
